# 간호사 근무 스케줄러

## 프로젝트 개요
병동 간호사(30명 규모) 3교대 근무 스케줄 관리 Windows 데스크탑 앱

## 기술 스택
- **UI**: PyQt5 (Windows/Mac 크로스플랫폼, PyInstaller로 .exe 패키징)
- **DB**: SQLite (schedule.db)
- **엑셀 출력**: openpyxl
- **알고리즘**: OR-Tools CP-SAT 최적화 솔버 (Google, ~0.2초/월)

## 실행 방법
```bash
cd /Users/hanjuan/study/schedule
source venv/bin/activate
python3 main.py
```

## Windows .exe 패키징
```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name "간호사스케줄러" main.py
# dist/간호사스케줄러.exe 생성됨
```

## 프로젝트 구조
```
schedule/
├── main.py              # 앱 진입점 + 메인 윈도우

├── scheduler.py         # 자동 스케줄 생성 알고리즘

    ├── nurse_tab.py     # 간호사 등록/수정/비활성화
    ├── request_tab.py   # 개인 휴무·근무 요청 입력

```

## 주요 기능

### 스케줄 탭
- 월별 달력 그리드 (간호사 × 날짜)
- 셀 더블클릭: D → E → N → O 순환
- 공휴일 헤더 표시 (빨간색)
- 자동 생성 / 저장 / 엑셀 출력
- 낮·저녁·밤·근무합 자동 집계

### 간호사 관리 탭
- 이름, 직책(수간호사/책임/일반), 입사일, 메모
- 비활성화 (삭제 대신 플래그 처리)

### 근무 요청 탭
- 월별 요청 그리드
- 셀 더블클릭: 빈칸 → 휴무 → 낮 → 저녁 → 밤 → 빈칸 순환

### 공휴일 탭
- 월별 공휴일 추가/수정/삭제
- 법정공휴일 자동 불러오기 (신정, 삼일절, 어린이날 등 8종)
- 스케줄 그리드 헤더에 공휴일명 표시

### 설정 탭
- 교대별 최소 인원 (낮/저녁/밤)
- 최대 연속 야간 일수
- 야간 종료 후 의무 휴무일

## 스케줄 알고리즘 제약 조건 (OR-Tools CP-SAT)

## 2026-07-03 — 실제 91병동 근무표 반영 (RAG 세션)

실제 7월 근무표 사진과 대조해 다음을 추가. 헤드리스 테스트(`/usr/bin/python3` 3.9 venv + ortools)로 전부 검증.

### 스케줄러 (scheduler.py)
- **전월 이월(월 경계 연속성)**: `xv()` 헬퍼로 전월 말일 근무를 offset(≤0) 상수로 주입.
  연속야간(C3)·야간후휴무(C4)·E→D(C8)·연속근무(C9/C13) 제약이 월 경계를 넘어 적용.
  전월 스케줄은 DB(`schedules`)에서 자동 로드. → 수작업 경계관리 자동화.
- **요일유형별 필요 인원**: 주말·공휴일 D/E/N 각 정확히 N명, 평일 N 정확히 N명(D/E는 최소).
  공휴일은 주말과 동일 취급.
- **RN/NA 직군 분리**: `generate()`가 RN/NA 풀을 나눠 각각 `_solve_pool()`로 풀고 병합.
  RN은 스킬믹스 적용, NA는 미적용·별도 인원(`na_*` 설정).
- **스킬믹스 소프트 제약 전환(C10/C11)**: 하드 제약이면 책임/숙련이 부족한 로스터에서
  스케줄이 아예 안 나옴(Infeasible). → 교대별 부족분(shortage)을 목적함수로 최소화해
  **생성은 항상 성공 + 가능한 최대 커버리지** 확보. 책임 부족에 가중치 3, 숙련 1.
  `generate(..., solve_seconds=12.0)`로 솔버 시간 상한 조정(소프트 목적함수는 최적 증명에
  시간이 걸려 상한이 필요). 신규 단독 방지(C12)는 하드 유지.
  - 실측: 시드 로스터(책임 5명)로 예전엔 Infeasible → 지금은 생성 성공,
    책임 커버리지 75/93(=80%, 책임 5명의 수학적 최대치), 숙련 93/93(100%).
