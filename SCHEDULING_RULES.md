# 간호사 근무표 자동생성 — 규칙 정의서

> 이 문서는 스케줄러(`scheduler.py`)가 근무표를 짤 때 적용하는 **모든 규칙**을 정리한 것이다.
> 규칙은 크게 **하드 규칙**(반드시 지켜짐)과 **소프트 규칙**(되도록 지키되 목적함수로 최소화)으로 나뉜다.
> 실제 값은 대부분 `settings` 테이블 설정으로 조정 가능하다([9. 설정값](#9-설정값-defaults) 참고).
> 마지막 갱신: 2026-07-03

---

## 1. 기본 개념

- **교대(shift)**: `D`(데이/낮) · `E`(이브닝/저녁) · `N`(나이트/밤) · `O`(오프/휴무)
- **직군(pool)**: **RN**(간호사) 과 **NA&HA**(간호조무사·보조인력) 를 **완전히 분리**해 각각 따로 편성 후 합친다.
  - RN 풀에만 스킬믹스·듀티 퍼스트 등 RN 전용 규칙이 적용된다.
- **엔진**: Google OR-Tools CP-SAT. 하드 규칙은 제약(constraint), 소프트 규칙은 목적함수(penalty) 최소화로 처리.
- **월 경계(전월 이월)**: 전월 말일 근무를 이어받아 연속·휴무 규칙이 달을 넘어 적용된다([8. 월 경계](#8-월-경계-처리-전월-이월) 참고).

## 2. 하드 규칙 (반드시 지켜짐)

| # | 규칙 | 내용 |
|---|------|------|
| **C1** | 하루 1교대 | 각 간호사는 매일 D/E/N/O 중 **정확히 하나** |
| **C2** | 교대별 필요 인원 | **주말·공휴일**: D/E/N **각 정확히 5명**(RN) · **평일**: N **정확히 5명**, D **≥5**, E **≥4**(RN). NA는 주말 2/2/1, 평일 N 1 |
| **C3** | 최대 연속 야간 | **3연속 야간까지** 허용, 4연속 금지 (전월 이월) |
| **C4** | 야간 후 의무 휴무 | 야간 블록이 끝나면 **2일 오프** 강제 (전월 이월) |
| **야간 텀** | 야간 블록 간격 | 야간 블록 시작 사이 **최소 5일 간격**(`night_gap`). 나이트 전담·야간불가자는 제외 |
| **C5** | 근무 요청 | 직원이 신청한 근무/오프(**바탕색 오프**)는 **그대로 반영** |
| **C6** | 야간 불가 | `no_night` 또는 **직책 갑계장**은 **야간(N) 배정 안 함** |
| **C7** | 월 최대 야간 | 1인당 월 야간 상한(`max_monthly_night`). 나이트 전담은 제외 |
| **C8** | 역방향 금지 | **E→D, E→휴→D, N→휴→D 금지** (생체리듬 보호, 전월 이월) |
| **C9** | 최대 연속 근무 | **5일 연속 근무까지** (6연속 금지). *월 내부는 하드, 월 경계는 소프트* |
| **C13** | 연속근무 후 휴무 | 5일 연속 근무 시 다음 **2일 오프** 강제. *월 내부 하드, 경계 소프트* |
| **전담** | 나이트 전담 | 전담자는 **밤/오프만**(D·E 금지), 야간은 **정확히 2연속(NN)**, 단독·3연속 금지 |

> **월 경계가 소프트인 이유**: 오프를 줄이려 월말에 근무가 몰리면, 경계 규칙을 하드로 두었을 때
> 다음 달 초를 대량 강제오프시켜 **다음 달이 아예 생성 불가**가 된다. 그래서 C9/C13의 *경계를 넘는 창만*
> 소프트(페널티)로 처리한다. 실측상 경계 위반은 대개 0건이며, 나더라도 NA 소수가 이음매에서 6연속 정도.

## 3. 소프트 규칙 (되도록 지킴 — 목적함수 최소화)

값이 클수록 더 강하게 지킨다(가중치). 여러 규칙이 상충하면 가중치로 우선순위가 정해진다.

| 규칙 | 가중치 | 내용 |
|------|:---:|------|
| **듀티 퍼스트** | **15** | 각 날 D/E/N에 **연차 8년+ '퍼스트'(책임자) 1명 이상** (`first_min_years`, 기본 8) |
| **프리셉터 일치** | **8** | 프리셉티는 프리셉터와 **매일 같은 교대** (근무표가 거의 같게) |
| **연차별 야간(비례)** | 3×티어 | **연차 높을수록 야간 적게**, 낮을수록 많이. 티어: 15년+ ×3, 8-14년 ×2, 3-7년 ×1, 0-2년 ×0 |
| **비전담 야간 상한** | 3 | 비전담 RN은 월 **야간 ~6개 이하**(`night_regular_cap`) |
| **오프 목표** | 3 | 월 오프를 **11개**(`monthly_off`)에 **위·아래로 수렴**. 신청 오프가 더 많으면 그만큼 인정 |
| **나이트 전담 목표** | 3 | 전담자 월 야간 **~15개**(`night_only_target`) |
| **월 경계 연속근무** | 25 | 월 경계를 넘는 C9/C13 위반 억제(하드 대체) |
| **스킬믹스(책임)** | 3 | 각 교대에 **책임간호사(등급) 1명 이상** (`min_charge_per_shift`) |
| **스킬믹스(숙련)** | 1 | 각 교대에 **숙련자(숙련+책임) 1명 이상** (`min_skilled_per_shift`) |
| **퐁당퐁당 방지** | 2 | **근무-휴-근무**(고립 휴무), **휴-근무-휴**(고립 근무) 억제 → 몰아서 근무/휴무 |
| **쓰리나이트 회피** | 2 | 3연속 야간(허용은 되나) **최대한 회피** → 1~2연속 선호 |
| **D→E→N 회피** | 3 | 데이-이브닝-나이트 **3일 연속 회전 회피** |
| **5일 연속근무 회피** | 3 | 5연속 근무(허용은 되나) **최대한 회피** → 4일 이하 블록 선호 |

> **스킬믹스는 왜 소프트인가**: 책임/숙련이 부족한 로스터에서 하드로 두면 아예 해가 없어진다.
> 부족분을 페널티로 최소화해 "**생성은 항상 성공 + 가능한 최대 커버리지**"를 보장한다.

## 4. 특수 간호사 유형

### 4-1. 갑계장 (야간 제외)
- **직책이 `갑계장`** 이면 자동으로 **야간(N) 배정 제외** (C6). 별도 야간불가 체크 불필요.
- 예(데모): **간호사01, 간호사19**.

### 4-2. 나이트 전담 (`night_only`)
- **밤(N)/오프(O)만** 근무 (D·E 배정 안 함).
- **NN-OO 패턴**: 야간은 **정확히 2연속**(단독 야간·3연속 없음), 월 **~15야간**.
- 예(데모): **간호사06, 간호사24**.
- 전담자는 오프 목표(11)·야간 텀·연차별 야간·비전담 상한 등에서 **제외**되고 전담 규칙만 따른다.

### 4-3. 프리셉터-프리셉티
- 프리셉티에게 **프리셉터(짝)** 를 지정하면 둘의 **근무표가 거의 같아지도록**(교대 일치) 유도.
- 현재 페어 (앞=프리셉터 → 뒤=프리셉티):
  - A팀 **간호사03 → 간호사04**
  - B팀 **간호사09 → 간호사10**
  - C팀 **간호사15 → 간호사16**
  - D팀 **간호사21 → 간호사20**

## 5. 직책 / 등급 체계

| 구분 | 값 | 근무 영향 |
|------|----|-----------|
| **직책(position)** | 갑계장 / 을계장 / 주임 / 대리 / 사원 | **갑계장 = 야간 제외**. 나머지는 현재 근무규칙에 직접 영향 없음 |
| **등급(level)** | 신규 / 일반 / 숙련 / 책임 | 스킬믹스(책임·숙련·신규단독방지)에 사용 |
| **연차(seniority)** | 입사연도/사번 앞4자리로 계산 | 듀티 퍼스트(10년+), 연차별 야간분배, 팀 밸런싱에 사용 |

> 신규 단독 교대 방지: 각 교대에 **비신규 1명 이상**(`prevent_new_only`, 소프트/하드는 스킬믹스와 함께).

---

## 6. 팀 편성 (연차 밸런싱)
- 팀(A~F)에 RN을 **연차·등급이 고르게** 배분 (탐욕 배정). NA는 팀 편성 제외.
- 팀 관리 탭의 **"⚖️ 연차 밸런싱 자동편성"** 버튼 → 미리보기 후 적용.

---

## 7. 답변/생성 파이프라인 (요약)
1. 활성 간호사 로드 → RN / NA 풀 분리
2. 전월 스케줄 자동 로드(전월 이월용)
3. 풀별로 CP-SAT 모델 구성 (하드 제약 + 소프트 목적함수)
4. 솔버 실행(직군별 시간 상한 `solve_seconds`, 기본 5초)
5. 결과 병합 → 저장/출력

---

## 8. 월 경계 처리 (전월 이월)
- 전월 말일 근무를 `day 0, -1, -2 …`(offset ≤0)로 모델에 **상수 주입**(`xv` 헬퍼).
- 덕분에 **연속야간(C3)·야간후휴무(C4)·역방향(C8)·연속근무(C9/C13)** 가 **달을 넘어** 적용된다.
- 전월 데이터는 `schedules` 테이블에서 자동 로드(첫 달은 이력 없이 동일 동작).

## 11. 실측 검증 (2026년 7월, RN 33 + NA 13)
- 하드 규칙 위반 **0건** (7·8월 및 월 경계)
- 오프 **전원 ~11개** / 퐁당퐁당 **≈0** / D→E→N **0건** / 5연속근무 **소수**
- 갑계장 야간 **0** / 나이트 전담 **NN-OO 유지** / 프리셉터 **31/31 일치**
- 연차별 야간: 저연차 최다(≈7), 고연차 최소(듀티 퍼스트로 밤 일부 담당)
- 듀티 퍼스트 커버(8년+): D·E·N **모두 100%** (후보 8명)

## 12. 알려진 트레이드오프 (규칙 상충)
1. **연차별 야간(시니어 적게)** vs **듀티 퍼스트(밤에도 10년+ 필요)** → 시니어 일부가 밤 퍼스트로 들어감.
2. **오프 11개** vs **야간 텀/5연속회피** → 근무 블록이 짧아져 오프가 11을 약간 넘길 수 있음.
3. **월 경계** → 다음 달 생성 가능성을 위해 경계 연속근무는 소프트(가끔 이음매 6연속).
4. **스킬믹스(밤 책임)** vs **연차별 야간** → 책임=시니어면 밤 책임 커버가 낮아질 수 있음.

> 이 상충들은 가중치([3. 소프트 규칙](#3-소프트-규칙-되도록-지킴--목적함수-최소화))로 우선순위를 조절한다.
